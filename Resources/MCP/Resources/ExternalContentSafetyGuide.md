# External Content Safety And Provenance

1. Clearly label externally generated text or JSON as assistant-authored support material, not source-of-truth engine data.
2. Keep prompts and generated content in repo-tracked or config-tracked locations when they affect shared workflows.
3. Do not copy vendor prompt libraries or proprietary tool text into UEBridgeMCP.
4. Record which provider, model, and input brief produced any accepted output.
5. Validate generated content against project conventions before applying it to assets or gameplay logic.
